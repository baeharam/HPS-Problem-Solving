Write-Up

* 내가 이해한 문제내용
    * 1,2,3의 합으로 1<= n <= 10 인 정수 n을 표현하는 조합수를 계산하는 프로그램

* 접근 방식
    * Base case인 1, 2, 3에 대해서 직접 구해보았다.
    * Case 4인 경우 1+3, 2+2, 3+1로 나타낼 수 있기 때문에 3을 만드는 경우, 2를 만드는 경우, 1을 만드는 경우를 모두 더하면 4를 만드는 경우를 만들수 있다고 가정했다.
    * 즉, f(n) = f(n-1) + f(n-2) + f(n-3) (n >= 4) 

* 어려웠던 점
    * Base case만 보고 규칙을 찾아내려해서 숫자를 분할한다는 생각을 하기 까지 오려 걸렸다.

* 복잡도
    * 시간복잡도: 1중 반복문 3개를 사용하기 때문에 O(3n) -> O(n)
    * 공간복잡도: DP를 위한 크기가 n인 배열을 할당하므로 O(n)
